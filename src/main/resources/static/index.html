Proyecto Web: Agenda de Salones de Eventos

Introducción

El proyecto Agenda de Salones de Eventos tiene como propósito desarrollar una aplicación web que permita gestionar reservas de salones para cumpleaños, reuniones y bodas. La aplicación mostrará disponibilidad, características de cada salón y permitirá la carga de documentos asociados.

Actualmente estamos en la fase de arquitectura, donde definimos la estructura del proyecto, los roles del equipo y la organización de carpetas antes de comenzar a programar.

Arquitectura del Proyecto

La arquitectura sigue el patrón multicapa, garantizando separación de responsabilidades, mantenibilidad y escalabilidad:

  • Controller: recibe solicitudes HTTP y expone los endpoints REST.
  • Service: contiene la lógica de negocio y validaciones.
  • Repository: maneja el acceso y consultas a la base de datos.
  • Model: define las entidades del dominio (Salón, Reserva, Usuario).
  • Resources: configuraciones de la aplicación y base de datos.
  • Test: pruebas unitarias e integración.
